`@adonis-agora/agent` shipped a focused, governed core first: a streaming agent loop, tool-calling with HITL, fail-closed authorization, quota and cost accounting, Lucid persistence, multi-agent delegation, and a testing kit. Several adjacent capabilities were designed for from day one (the SPIs left the seams) but built afterward. The **parity-sync** release closes most of those gaps.

<Callout type="info" title="What this page tracks">
  Honesty about scope: what's real now (with links), and what's still intentionally out. If a capability moved from "deferred" to "shipped", it's because the code and tests landed — not a wishlist.
</Callout>

## Shipped since the initial core

### The durable runner

The headline. The [agent loop](/docs/agent/concepts/agent-loop) was written against a hooks seam precisely so the *same* body runs in-process or durably — and now `durable: true` runs each turn as a replay-safe [`@adonis-agora/durable`](/docs/durable) workflow: memoized `localStep` checkpoints, HITL approval as a `waitForSignal` suspend that survives restarts, and delegation as a `ctx.child` run. It degrades gracefully to inline when the peer isn't wired. See [The durable runner](/docs/agent/durability/durable-runner).

### RAG & retrieval

A full retrieval stack: the `Retriever` / `EmbeddingProvider` / `Reranker` SPIs, an in-memory cosine retriever and a production [pgvector store](/docs/agent/retrieval/pgvector), always-on "inject" retrieval that folds cited passages into the system prompt, and [media auto-ingestion](/docs/agent/retrieval/media-ingestion) that indexes uploads off an `agora:media:upload.complete` event. See [RAG & retrieval](/docs/agent/retrieval/rag).

### The governance read-model, dashboard & Telescope tab

`governanceQueries.lucid()` is a Lucid read-model over the persisted tables — spend, usage trend, run lifecycle, per-tool stats, reliability, and a cross-thread approvals inbox — exposed as `/agent/governance/*` routes. The standalone [`@adonis-agora/agent-dashboard`](/docs/agent/governance/dashboard) SPA and the [Telescope "Agent" tab](/docs/agent/governance/telescope) both consume it. See [Governance read-model](/docs/agent/governance/read-model).

### `agent_run` lifecycle tracking

Both runners record each run to an `agent_run` table (`running → completed | failed | cancelled`, with step/token counts and a `durable` flag), back-referenced by messages, tool calls, and usage — the spine of the run-lifecycle governance surfaces. See [Governance read-model](/docs/agent/governance/read-model#run-lifecycle-tracking).

### The authz (Bouncer) adapter

`authzToolAuthorizer` swaps the ADMIN-only role gate for ability-based authorization through [`@adonis-agora/authz`](/docs/authz) — checking each tool's declared `ability`, tenant-scoped and fail-closed (no ADMIN fallback). See [Authz (Bouncer) adapter](/docs/agent/governance/authz-bouncer).

### The `data` governed-SQL satellite

`dataTool` gives the agent a single fail-closed read-only SQL tool: one validated `SELECT`, a required table allow-list, tenant-scope rewriting, `LIMIT` injection, and result truncation. See [Governed SQL](/docs/agent/authoring/data-satellite).

### Multimodal attachments

Images and PDFs attach to a message and reach a vision-capable model as native parts, behind an attachment-staging seam and an optional `POST /agent/attachments` upload route. See [Attachments & multimodal](/docs/agent/authoring/attachments).

### Transient tool retries & tool filters

A tool's own invocation retries in place on a classified-transient DB error (deadlock / lock-wait / serialization), replay-safe and on by default; business failures stay one-shot. See [Transient tool retries](/docs/agent/authoring/transient-retry).

### Redis multi-replica streaming

`tokenSinks.redis(...)` fans a run's tokens across replicas so any pod serves any run's SSE stream, with the envelope byte-identical — plus an actor-directory seam for read-side identity labels. See [Multi-replica streaming](/docs/agent/redis-streaming).

## Still deferred

### Embedding & reranker adapters

The `EmbeddingProvider` and `Reranker` SPIs and both bundled retrievers ship, but a first-party `aiSdkEmbedding` adapter (over the AI SDK's `embedMany`) and provider rerankers (Cohere/Voyage) are not built yet — wire your own or use the testing doubles. See [RAG & retrieval](/docs/agent/retrieval/rag).

### An Agora-scoped React package

The framework-agnostic `@dudousxd/nestjs-agent-react` client (`useAgentChat` / `AgentChatTransport`) already works **unchanged** against these routes, because the SSE contract is preserved byte-for-byte — see [Streaming & HTTP](/docs/agent/streaming-and-http#consuming-from-react). An **Agora-scoped republish** (matching branding and imports) is still on the list, but it's ergonomics, not a capability gap.

## Deliberately dropped

### Build-time codegen of tool wrappers

The reference NestJS implementation used a codegen step. On AdonisJS that's unnecessary and intentionally **not** ported: the package registers an **Assembler `init` hook** that generates the typed `app/agent_tools` barrel at build/dev time (with a runtime `readdir` scan as the fallback), which is the idiomatic Adonis mechanism. See [Tools → discovery](/docs/agent/authoring/tools#discovery).
